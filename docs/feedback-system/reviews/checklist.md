# Review Checklist

## Pre-Review
- [ ] Verify the submission follows project guidelines
- [ ] Check for duplicate or related submissions
- [ ] Confirm all required fields are filled

## Content Review
- [ ] Technical accuracy is verified
- [ ] Code examples are correct and runnable
- [ ] Documentation is clear and complete
- [ ] Links and references are valid

## Quality Review
- [ ] Follows project style and conventions
- [ ] No sensitive information exposed
- [ ] Appropriate error handling included
- [ ] Tests are present and passing (if applicable)

## Feedback Response
- [ ] Acknowledge receipt of feedback
- [ ] Provide estimated timeline for review
- [ ] Communicate decisions clearly
- [ ] Offer alternatives if feedback cannot be implemented

## Post-Review
- [ ] Update feedback status in tracking system
- [ ] Close or forward to appropriate team
- [ ] Archive review notes for future reference
